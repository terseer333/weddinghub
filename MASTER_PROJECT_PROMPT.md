# WeddingHub Master Project Prompt

I am building a production-quality full-stack wedding management platform called **WeddingHub**. The backend is written in Go (Golang) using the standard `net/http` package and follows a simple structure with `cmd/main.go`, `handlers`, `models`, `pages`, and an `uploads` directory. The frontend is built with HTML, CSS, and Vanilla JavaScript without any framework. The current application already has working user authentication (signup and login), event creation, event retrieval, wedding invitation pages, RSVP submission, RSVP listing, dashboard statistics, image upload support, CORS handling, and a responsive luxury-themed UI. The backend currently uses in-memory slices for storing users, events, and RSVPs, but the long-term goal is to migrate to a real database such as PostgreSQL. The application already exposes routes including `/signup`, `/login`, `/users`, `/events`, `/events/all`, `/event`, `/rsvp`, `/rsvp/all`, and `/upload`, and uploaded files are served from the `/uploads` directory. The dashboard displays wedding statistics, event cards, and navigation to invitations and RSVP management. The invitation page contains a hero section, countdown timer, venue information, love story section, gallery section, and RSVP call-to-action. The create-event page allows users to create weddings and upload pre-wedding images. The RSVP page allows guests to confirm attendance and leave messages. The UI should continue using a premium romantic wedding aesthetic with elegant typography, gradients, soft shadows, responsive layouts, beautiful cards, image galleries, countdown widgets, and a modern SaaS-style dashboard. Going forward, all improvements should build on the existing codebase rather than recreating completed features. Remaining goals include connecting uploaded images directly to events, displaying uploaded images dynamically in the invitation hero slider and dashboard cards, supporting multiple gallery images per event, implementing event update and delete functionality, adding image previews and drag-and-drop uploads, creating richer dashboard analytics and RSVP statistics, adding guest management tools, supporting public shareable invitation links, integrating maps and wedding schedules, implementing secure session management or JWT authentication, migrating from in-memory storage to PostgreSQL, creating an admin panel for managing users, events, and RSVPs, improving mobile responsiveness, and ultimately turning WeddingHub into a professional wedding website builder comparable to platforms like The Knot, WeddingWire, Joy, and Zola. Whenever continuing this project, first analyze the existing code, preserve completed functionality, explain changes clearly, provide complete production-ready code where necessary, maintain consistency across frontend and backend, and always focus on finishing the application as a real-world deployable product rather than a tutorial or prototype.
